Thank you for choosing to contribute! This project welcomes first-time open-source contributors.

Getting started

- Install dependencies: `npm install`
- Run the dev server: `npm run dev`

How to pick a task

- See `GOOD_FIRST_ISSUES.md` for curated beginner-friendly tasks.
- Look for issues labeled `good first issue` or `help wanted`.

Working on a change

- Create a branch named `fix/short-description` or `feat/short-description`.
- Make small, focused commits with clear messages.
- Run the dev server and verify your change locally before submitting a PR.

Submitting a PR

- Open a PR that references the issue (if any) and describes what you changed.
- Keep the PR title concise and start with the type (e.g., `Fix:` or `Feat:`).
- Add screenshots for visual changes.

Code style and tests

- Follow the existing code style and TypeScript patterns in the repo.
- If you add functionality, include tests when reasonable.

Need help?

- Comment on the issue you're working on if you need guidance. Maintainers will help you get unstuck.
